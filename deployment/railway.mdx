## Overview

Railway is a modern deployment platform that makes it easy to deploy Spring Boot applications with PostgreSQL databases. This guide walks you through deploying the Q_SP API to Railway.

<Note>
  The Q_SP API is pre-configured for Railway deployment with environment variable support for database connection and server port.
</Note>

## Prerequisites

Before deploying, ensure you have:

- A [Railway account](https://railway.app/) (free tier available)
- Your Q_SP API source code in a Git repository (GitHub, GitLab, or Bitbucket)
- Java 21 and Maven configured in your project

## Deployment Steps

<Steps>
  <Step title="Create a New Project">
    1. Log in to your Railway dashboard
    2. Click **New Project**
    3. Select **Deploy from GitHub repo**
    4. Authorize Railway to access your repositories
    5. Select your Q_SP API repository
  </Step>

  <Step title="Add PostgreSQL Database">
    Railway makes it easy to add a PostgreSQL database:
    
    1. In your project dashboard, click **New**
    2. Select **Database** → **Add PostgreSQL**
    3. Railway will automatically provision a PostgreSQL instance
    4. The database will be configured with automatic backups
    
    <Note>
      Railway automatically creates the `PGHOST`, `PGPORT`, `PGDATABASE`, `PGUSER`, and `PGPASSWORD` environment variables when you add PostgreSQL.
    </Note>
  </Step>

  <Step title="Configure Environment Variables">
    The application is already configured to use Railway's PostgreSQL environment variables. Railway automatically provides:
    
    - `PGHOST` - Database host
    - `PGPORT` - Database port (default: 5432)
    - `PGDATABASE` - Database name
    - `PGUSER` - Database username
    - `PGPASSWORD` - Database password
    
    Railway also provides:
    - `PORT` - The port your application should listen on
    
    <Warning>
      Do not manually set these variables - Railway configures them automatically when you add the PostgreSQL service.
    </Warning>
  </Step>

  <Step title="Configure Build Settings">
    Railway should automatically detect your Spring Boot application. Verify the build configuration:
    
    1. Go to your service settings
    2. Check that the **Build Command** is set to:
    ```bash
    mvn clean package -DskipTests
    ```
    
    3. Check that the **Start Command** is:
    ```bash
    java -jar target/Q_SP-0.0.1-SNAPSHOT.jar
    ```
    
    <Note>
      Railway auto-detects Maven projects and usually configures these commands automatically.
    </Note>
  </Step>

  <Step title="Initialize Database Schema">
    Since the application uses `spring.jpa.hibernate.ddl-auto=none`, you need to manually create the database schema:
    
    1. Click on your PostgreSQL service
    2. Go to the **Connect** tab
    3. Click **Open PostgreSQL Console** or use the provided connection URL
    4. Run the SQL schema creation scripts
    
    See [Database Setup](/deployment/database-setup) for the complete schema.
  </Step>

  <Step title="Deploy">
    1. Railway will automatically trigger a deployment after connecting your repository
    2. Monitor the deployment logs in the Railway dashboard
    3. Once deployed, Railway provides a public URL for your API
    4. Test your API at `https://your-app.railway.app/api/products`
  </Step>
</Steps>

## Configuration Details

### Port Configuration

The application is configured to use Railway's `PORT` environment variable:

```properties
server.port=${PORT:8080}
```

This means:
- On Railway, it uses the `PORT` variable provided by the platform
- Locally, it defaults to port 8080

### Database Connection

The `application.properties` file is pre-configured for Railway:

```properties
spring.datasource.url=jdbc:postgresql://${PGHOST}:${PGPORT}/${PGDATABASE}
spring.datasource.username=${PGUSER}
spring.datasource.password=${PGPASSWORD}
spring.datasource.driver-class-name=org.postgresql.Driver
```

## Monitoring and Logs

<Card title="View Application Logs" icon="scroll">
  Access real-time logs in the Railway dashboard:
  1. Select your service
  2. Click on the **Logs** tab
  3. Monitor application startup, SQL queries, and errors
</Card>

The application is configured to show SQL queries:

```properties
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
logging.level.org.hibernate.SQL=DEBUG
```

<Warning>
  In production, consider disabling SQL logging by setting `spring.jpa.show-sql=false` to improve performance and reduce log noise.
</Warning>

## Custom Domain

To add a custom domain:

1. Go to your service settings
2. Navigate to **Domains**
3. Click **Add Custom Domain**
4. Follow Railway's instructions to configure DNS

## Troubleshooting

### Build Fails

- Ensure Java 21 is specified in your `pom.xml`
- Check that all dependencies are properly configured
- Verify Railway is using the correct build command

### Database Connection Errors

- Verify the PostgreSQL service is running
- Check that the database environment variables are set
- Ensure the database schema is created (ddl-auto is set to `none`)

### Application Won't Start

- Check the logs for startup errors
- Verify the `PORT` environment variable is being used correctly
- Ensure the JAR file name matches the start command
