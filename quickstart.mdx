## Get your API up and running

This guide will walk you through setting up the Q_SP API locally, from cloning the repository to making your first API call.

<Steps>

### Clone the repository

Start by cloning the Q_SP API repository from GitHub:

```bash
git clone https://github.com/JhonnyDiaz4753/Api_Q_SP.git
cd Api_Q_SP
```

### Set up PostgreSQL database

The Q_SP API requires a PostgreSQL database. You can either:

- Install PostgreSQL locally ([Download PostgreSQL](https://www.postgresql.org/download/))
- Use a cloud provider like [Railway](https://railway.app) or [Neon](https://neon.tech)

**Create a database:**

```sql
CREATE DATABASE q_sp_db;
```

You'll need to create the following tables:

```sql
CREATE TABLE categories (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255),
    description TEXT,
    active BOOLEAN,
    icon VARCHAR(255)
);

CREATE TABLE products (
    id_products SERIAL PRIMARY KEY,
    name_products VARCHAR(255),
    description TEXT,
    price INTEGER,
    imge_url VARCHAR(255),
    active BOOLEAN,
    category_id INTEGER REFERENCES categories(id)
);
```

<Note>
The application uses `spring.jpa.hibernate.ddl-auto=none`, so you must create the database schema manually.
</Note>

### Configure environment variables

Set the following environment variables for database connection:

```bash
export PGHOST=localhost
export PGPORT=5432
export PGDATABASE=q_sp_db
export PGUSER=your_username
export PGPASSWORD=your_password
export PORT=8080
```

Or create a `.env` file in your project root (you'll need to load these manually or use a library).

<Warning>
Never commit your `.env` file or expose database credentials in your code.
</Warning>

### Build and run the application

The Q_SP API uses Maven for dependency management. Build and run using the included Maven wrapper:

**On Linux/macOS:**

```bash
./mvnw clean install
./mvnw spring-boot:run
```

**On Windows:**

```bash
mvnw.cmd clean install
mvnw.cmd spring-boot:run
```

You should see output indicating the application has started:

```
Started QSpApplication in X.XXX seconds
```

The API will be available at `http://localhost:8080`.

### Make your first API call

Test the API by fetching all categories:

```bash
curl http://localhost:8080/categories
```

You should receive a JSON response with your categories:

```json
[
  {
    "id": 1,
    "name": "Bebidas",
    "description": "Todas las bebidas",
    "active": true,
    "icon": "drink-icon.png"
  }
]
```

**Try fetching all products:**

```bash
curl http://localhost:8080/products
```

**Or get products by category:**

```bash
curl http://localhost:8080/products/category/1
```

</Steps>

## Access API documentation

The Q_SP API includes Springdoc OpenAPI integration. Once your application is running, access the interactive API documentation:

- **Swagger UI**: [http://localhost:8080/swagger-ui.html](http://localhost:8080/swagger-ui.html)
- **OpenAPI JSON**: [http://localhost:8080/v3/api-docs](http://localhost:8080/v3/api-docs)
